Features Used for Phishing Domain Detection

There are many algorithms and a wide variety of data types used for phishing detection in academic literature and commercial products. A phishing URL and the corresponding page have several features that differentiate them from legitimate URLs.


For example, an attacker can register long and confusing domain names to hide the actual domain name (Cybersquatting, Typosquatting). In some cases, attackers can use direct IP addresses instead of using the domain name.

URL-Based Features

URL analysis is the first step in determining whether a website is phishing or not. As mentioned before, URLs of phishing domains have some distinctive characteristics. Some of the URL-Based Features include:

  • Digit count in the URL
  • Total length of the URL
  • Checking for typosquatting (e.g., google.com â†’ goggle.com)
  • Checking for the inclusion of legitimate brand names (e.g., apple-icloud-login.com)
  • Number of subdomains in the URL
  • Whether the Top-Level Domain (TLD) is commonly used

Domain-Based Features

The purpose of phishing domain detection is to identify phishing domain names. Therefore, passive queries related to the domain name can provide useful information. Some useful Domain-Based Features include:

  • Whether the domain name or its IP address appears in blacklists of well-known reputation services
  • The number of days since the domain was registered
  • Whether the registrant name is hidden

Page-Based Features

Page-Based Features utilize information about pages from reputation ranking services. Some of these features provide insights into the reliability of a website, such as:

  • Global PageRank
  • Country PageRank
  • Position in the Alexa Top 1 Million Sites
  • Other Page-Based Features provide information about user activity on the target site, including:
    • Estimated number of visits for the domain on a daily, weekly, or monthly basis
    • Average page views per visit
    • Average visit duration
    • Web traffic share per country
    • Count of references from social networks to the given domain
    • Domain category
    • Similar websites

Content-Based Features

Obtaining these features requires an active scan of the target domain. Page contents are analyzed to detect whether the domain is used for phishing. Some processed information includes:

  • Page Titles
  • Meta Tags
  • Hidden Text
  • Text in the Body
  • Images
    By analyzing this information, we can gather insights such as:
  • Whether login is required on the website
  • Website category
  • Information about the audience profile
